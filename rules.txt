## 0. 🧭 THE "ANTI-MONOTONY" MANIFESTO

> **`Symmetry is safe. Asymmetry is intentional. Monotony is the enemy.`**

### AI Default Habit (❌ BLOCKED):
```
❌ flex-col untuk seluruh halaman
❌ grid-cols-1/2/3 dengan equal width
❌ Centered headers dengan margin auto
❌ Pure white/black backgrounds tanpa depth
❌ Static vertical scroll tanpa spatial rhythm
❌ Standard navbar + hero + cards + footer template
```

### Professional Editorial Standard (✅ REQUIRED):
```
✅ Off-grid typography dengan overlapping & negative space
✅ Asymmetric Bento Grids dengan variable spans & contextual hierarchy
✅ Parallax, sticky-sections, horizontal reveals untuk storytelling
✅ Subtle gradients, noise textures, layering untuk depth perception
✅ Typographic-first heros dengan visual weight, bukan hanya "title + subtitle"
✅ Scroll-driven animations dengan view-timeline & CSS containment
```

### Quality Definition (Universal Web)
| ✅ Harus Ada | 🚫 Tidak Perlu Day-1 |
|-------------|---------------------|
| Layout intentionally asymmetric di ≥2 section | 100% pixel-perfect tanpa user testing |
| Typography sebagai structural element (bukan hanya text) | Animasi kompleks di setiap interaksi |

## 1. 🚨 ANTI-HALLUCINATION & SCOPE PROTOCOL (WEB-SPECIFIC)

### 🔒 Scope Lock & Boundary Lock (WAJIB sebelum coding)
```markdown
🔒 WILL BUILD:
- [Layout Pattern A] – business/UX reason (1 kalimat)
- [Component B] – purpose & target screen (1 kalimat)  
- [Interaction C] – user value (1 kalimat)

🚫 WON'T BUILD:
- [Pattern X] – scope creep / Tier 3+ / kompleksitas belum perlu
- [Library Y] – premature abstraction / bloat risk

📏 SIZE ESTIMATION:
- Layout A: ~XXX lines | Component B: ~XXX lines | Total artifact: <1000 lines
```

### 📏 Size Police (Web-Adjusted)
| Artifact | Hard Limit | Soft Buffer | Action if Exceeded |
|----------|-----------|-------------|-------------------|
| Page Component | 500 lines | 550 | Extract sections to feature components |
| Layout Pattern | 300 lines | 330 | Extract to `shared/layouts/` |
| Custom Hook | 250 lines | 280 | Extract focused hooks |
| CSS/Tailwind String | 40 classes | - | Extract to `@layer`, `cva`, atau CSS vars |
| Single Output Artifact | 1000 lines | - | Chunk into route segments / lazy load |

### 🔍 Logic Police (Web Context)
1. **Is this interaction strictly required?** (Uncertain → ❌ omit)
2. **>3 conditional layout branches?** → Extract to layout component atau custom hook
3. **Scroll/viewport dependent?** → Use `IntersectionObserver` / CSS `view-timeline`, bukan JS scroll listener
4. **SEO-critical content?** → Ensure server-rendered, not client-hydrated only

### 🛡️ Dependency Policy (Web Whitelist)
| Status | Libraries | Notes |
|--------|-----------|-------|
| ✅ Core (Wajib) | React 18+/Next.js 14+, Tailwind v3/v4, TypeScript strict | Baseline stack |
| ✅ Recommended | `clsx`/`tailwind-merge`, `framer-motion` (scroll animations ONLY), `@react-spring/web` (parallax), `lenis` (smooth scroll) | Use sparingly, justify each |
| ⚠️ Conditional | `three.js`/`@react-three/fiber` (3D ONLY if critical), `gsap` (complex timelines), `embla-carousel` (horizontal scroll) | Must have business justification |
| 🚫 Banned | `bootstrap`, `jquery`, `slick-carousel`, `owl.carousel`, inline `style={{}}` (kecuali dynamic `transform/opacity`), `lodash` | Bloat / anti-pattern |

### 👃 Web Code Smell Detector
```
❌ // TODO: add animation later → HALLUCINATION. Implement or delete.
❌ className="flex flex-col items-center justify-center min-h-screen" → MONOTONY. Break the grid.
❌ All cards same height/width → LAZY DESIGN. Use variable spans.
❌ Hardcoded breakpoints (sm:, md:) tanpa container queries → NOT RESPONSIVE.
❌ Scroll listener tanpa throttle/debounce → PERFORMANCE KILLER.
❌ Image tanpa width/height/aspect-ratio → CLS RISK.
❌ Text hardcoded di JSX tanpa i18n key → NOT UNIVERSAL.
❌ Z-index magic numbers (999, 9999) → LAYERING CHAOS. Use CSS layers.
❌ Animation tanpa prefers-reduced-motion → A11y VIOLATION.
❌ Console.log di production build → CLEANUP BEFORE SHIPPING.
```

### C. Advanced Layout Primitives Dictionary
| Pattern | CSS/Tailwind Strategy | Use Case | Example Class |
|---------|----------------------|----------|--------------|
| **Sticky Stacking** | `sticky top-0 z-30` + `-[margin]` for overlap | Scroll-aware sections, layered storytelling | `className="sticky top-0 z-30 -mt-12"` |
| **Horizontal Scroll Section** | `flex overflow-x-auto snap-x snap-mandatory` + `snap-center` | Galleries, testimonials, feature highlights | `className="flex overflow-x-auto snap-x pb-8"` |
| **Parallax Reveal** | `will-change-transform` + `data-scroll-speed` (Lenis) | Hero images, background layers | `className="will-change-transform" data-scroll-speed="0.5"` |
| **Container Query Layout** | `@container` + `@[300px]:grid-cols-2` | Component-level responsiveness | `className="@container"` + child `@[400px]:flex-row` |
| **CSS Subgrid Alignment** | `grid-template-columns: subgrid` | Nested grids that align to parent | `className="grid grid-cols-subgrid col-span-6"` |
| **View-Triggered Animation** | `animation-timeline: view()` + `animation-range` | Scroll-driven reveals, progress indicators | `className="animate-[fadeInUp_0.6s_ease] [animation-timeline:view()]"` |

### D. Responsive Rearrangement Rules (NOT Just Stacking)
```
📱 Mobile (<768px):
- Horizontal scrollers for card lists (reduce vertical height)
- Typography scales via clamp(), not breakpoints
- Touch targets minimum h-14 (56px)
- Navigation transforms to bottom sheet or floating dock

🖥️ Desktop (>1024px):
- Asymmetric grids with intentional white space
- Hover states for interactive elements
- Multi-column text with drop caps for editorial content
- Parallax/scroll effects enabled (with reduced-motion respect)

🔄 Rearrangement Logic:
- Use CSS `order` + `grid-template-areas` for visual reordering
- Keep DOM order logical for accessibility/SEO
- Test keyboard navigation after rearrangement
```

## 7. ⚡ PERFORMANCE, ACCESSIBILITY & DEVOPS

### A. Core Web Vitals 2025 Targets
| Metric | Target | Fix Strategy |
|--------|--------|--------------|
| LCP | `< 2.0s` | Preload hero image, inline critical CSS, defer non-critical JS |
| CLS | `< 0.05` | Reserve space for images/media, font `swap` + size lock, avoid dynamic injection above content |
| INP | `< 200ms` | Debounce inputs, use CSS transitions over JS, web workers for heavy calc |
| TBT | `< 200ms` | Code split routes, lazy load below-fold content, minimize main thread work |

### B. Accessibility (WCAG 2.2 AA + Editorial Enhancements)
```
✅ Focus-visible:ring-4 on all interactive elements (keyboard nav)
✅ aria-label for icon-only buttons, decorative images have alt=""
✅ Skip link: <a href="#main" class="sr-only focus:not-sr-only fixed top-4 start-4 z-50 bg-emerald-600 text-white px-4 py-2 rounded-lg">Skip to content</a>
✅ Color contrast ≥ 4.5:1 (text), ≥ 3:1 (large text/UI)
✅ Reduced motion: @media (prefers-reduced-motion: reduce) { * { animation: none !important; transition: none !important; } }
✅ Text resizing: Layout does not break at 200% zoom
```
